Edit on GitHub

QWeb - это основной движок шаблонов, используемый Odoo [#othertemplates] _. Это механизм шаблонов XML [#genshif] _ и используется в основном для генерации HTML фрагментов и страниц.

Шаблонные директивы указываются как атрибуты XML с префиксом `` t-, например `` t-if для Conditionals, с элементами и другими атрибутами, отображаемыми напрямую.

Чтобы избежать визуализации элемента, элемент-заполнитель `` <t> `` также доступен, который выполняет свою директиву, но не генерирует никакого вывода сам по себе:

<t t-if="condition">
    <p>Test</p>
</t>

приведет к

<p>Test</p>

если `` условие`` истинно, но

<div t-if="condition">
    <p>Test</p>
</div>

приведет к

<div>
    <p>Test</p>
</div>

Вывод данных

QWeb имеет первичную директиву вывода, которая автоматически выводит HTML из своего содержимого, ограничивая риски XSS при отображении пользовательского содержимого: `` esc``.

`` esc`` принимает выражение, оценивает его и печатает содержимое

<p><t t-esc="value"/></p>

отображается со значением `` value``, установленным в `` 42``, приводит к

<p>42</p>

Есть еще одна выходная директива `` raw``, которая ведет себя так же, как соответственно `` esc``, но * не экранирует HTML-выход *. Может быть полезно отобразить отдельно созданную разметку (например, из функций) или уже очищенную предоставленную пользователем разметку.

Conditionals

QWeb имеет условную директиву `` if``, которая оценивает выражение, заданное как значение атрибута:

<div>
    <t t-if="condition">
        <p>ok</p>
    </t>
</div>

Элемент отображается, если условие истинно:

<div>
    <p>ok</p>
</div>

но если условие ложно, оно удаляется из результата:

<div>
</div>

Условный рендеринг применяется к носителю директивы, который не обязательно должен быть `` <t> ``

<div>
    <p t-if="condition">ok</p>
</div>

даст те же результаты, что и в предыдущем примере.

Также доступны дополнительные условные директивы ветвления `` t-elif`` и `` t-else``

<div>
    <p t-if="user.birthday == today()">Happy birthday!</p>
    <p t-elif="user.login == 'root'">Welcome master!</p>
    <p t-else="">Welcome!</p>
</div>

Loops

QWeb имеет итерационную директиву `` foreach``, которая принимает выражение, возвращающее коллекцию для повторения, и второй параметр `` t-as``, предоставляющий имя, используемое для «текущего элемента» итерации

<t t-foreach="[1, 2, 3]" t-as="i">
    <p><t t-esc="i"/></p>
</t>

будет отображаться как

<p>1</p>
<p>2</p>
<p>3</p>

Подобно условиям, `` foreach`` применяется к элементу, содержащему атрибут директивы, и

<p t-foreach="[1, 2, 3]" t-as="i">
    <t t-esc="i"/>
</p>

эквивалентно предыдущему примеру.

`` foreach`` может выполнять итерации в массиве (текущий элемент будет текущим значением) или в отображении (текущий элемент будет текущим ключом). Итерирование по целому числу (эквивалентно итерации по массиву между 0 включительно и предоставленным целочисленным исключением) все еще поддерживается, но не рекомендуется.

В дополнение к имени, передаваемому через `` t-as``, `` foreach`` предоставляет несколько других переменных для различных точек данных:

Предупреждение

`` $ as`` будет заменено именем, переданным в `` t-as``

: samp: {$ as} _all (устарело)

объект перебирается

Примечание

Эта переменная доступна только в JavaScript QWeb, но не в Python.

: Сэмпл: {$}, как _value
текущее значение итерации, идентичное `` $ as`` для списков и целых чисел, но для отображений оно предоставляет значение (где `` $ as`` предоставляет ключ)
: Сэмпл: {$}, как _index
текущий индекс итерации (первый элемент итерации имеет индекс 0)
: Сэмпл: {$}, как _size
размер коллекции, если она доступна
: Сэмпл: {$}, как _first
является ли текущий элемент первым в итерации (эквивалентно: samp: {$ as} _index == 0)
: Сэмпл: {$}, как _last
является ли текущий элемент последним в итерации (эквивалентно: samp: {$ as} _index + 1 == {$ as} _size), требует ли размер итерируемого быть доступным
: samp: {$ as} _parity (устарело)
либо `` 'четное' `` или `` нечетное ``, четность текущего раунда итерации
: samp: {$ as} _even (устарело)
логический флаг, указывающий, что текущий раунд итерации находится на четном индексе
: samp: {$ as} _odd (устарело)
логический флаг, указывающий, что текущий раунд итерации находится на нечетном индексе

Эти дополнительные переменные и все новые переменные, созданные в `` foreach``, доступны только в области `` foreach``. Если переменная существует вне контекста `` foreach``, значение копируется в конце foreach в глобальный контекст.

<t t-set="existing_variable" t-value="False"/>
<!-- existing_variable now False -->

<p t-foreach="[1, 2, 3]" t-as="i">
    <t t-set="existing_variable" t-value="True"/>
    <t t-set="new_variable" t-value="True"/>
    <!-- existing_variable and new_variable now True -->
</p>

<!-- existing_variable always True -->
<!-- new_variable undefined -->

атрибуты

QWeb может вычислять атрибуты на лету и устанавливать результат вычисления на выходном узле. Это делается с помощью директивы `` t-att`` (attribute), которая существует в 3 различных формах:

: Сэмпл: т-ATT - {$ имя}

создается атрибут с именем `` $ name``, значение атрибута оценивается, а результат устанавливается как значение атрибута

<div t-att-a="42"/>

будет отображаться как

<div a="42"></div>
: Сэмпл: т-ATTF - {$ имя}

такой же, как и предыдущий, но параметром является: term: format string, а не просто выражение, часто полезное для смешивания литеральной и нелитеральной строки (например, classes)

<t t-foreach="[1, 2, 3]" t-as="item">
    <li t-attf-class="row {{ (item_index % 2 === 0) ? 'even' : 'odd' }}">
        <t t-esc="item"/>
    </li>
</t>

будет отображаться как

<li class="row even">1</li>
<li class="row odd">2</li>
<li class="row even">3</li>
: SAMP: т-Att = mapping

если параметр является отображением, каждая пара (ключ, значение) генерирует новый атрибут и его значение

<div t-att="{'a': 1, 'b': 2}"/>

будет отображаться как

<div a="1" b="2"></div>
: SAMP: т-Att = pair

если параметр является парой (кортеж или массив из 2 элементов), первым элементом пары является имя атрибута, а вторым элементом является значение

<div t-att="['a', 'b']"/>

будет отображаться как

<div a="b"></div>

установка переменных

QWeb позволяет создавать переменные из шаблона, запоминать вычисления (использовать их несколько раз), давать фрагменту данных более четкое имя, …

Это делается с помощью директивы `` set``, которая принимает имя создаваемой переменной. Значение для установки может быть предоставлено двумя способами:

  • атрибут `` t-value``, содержащий выражение, и результат его оценки будет установлен

    <t t-set="foo" t-value="2 + 1"/>
<t t-esc="foo"/>

    напечатает `` 3``

  • если атрибут `` t-value`` отсутствует, тело узла отображается и устанавливается как значение переменной

    <t t-set="foo">
    <li>ok</li>
</t>
<t t-esc="foo"/>

    сгенерирует `` &lt;li&gt; ok &lt;/ li&gt; `` (содержимое экранируется, поскольку мы использовали директиву `` esc``)

    Примечание

    использование результата этой операции является значительным примером использования директивы `` raw``.

вызов суб-шаблонов

Шаблоны QWeb можно использовать для рендеринга верхнего уровня, но их также можно использовать из другого шаблона (чтобы избежать дублирования или дать имена частям шаблонов) с помощью директивы `` t-call``

<t t-call="other-template"/>

Это вызывает именованный шаблон с контекстом выполнения родителя, если `` other_template`` определено как

<p><t t-value="var"/></p>

вызов выше будет обработан как `` <p/> `` (без контента), но

<t t-set="var" t-value="1"/>
<t t-call="other-template"/>

будет отображаться как `` <p> 1 </p> ``.

Однако проблема заключается в том, чтобы быть видимым снаружи `` t-call``. В качестве альтернативы, содержимое, установленное в теле директивы `` call``, будет оценено * перед * вызовом суб-шаблона и может изменить локальный контекст

<t t-call="other-template">
    <t t-set="var" t-value="1"/>
</t>
<!-- "var" does not exist here -->

Тело директивы `` call`` может быть произвольно сложным (не только директивами `` set``), и его визуализированная форма будет доступна в вызываемом шаблоне как магическая переменная `` 0``

<div>
    This template was called with content:
    <t t-raw="0"/>
</div>

вызывается таким образом:

<t t-call="other-template">
    <em>content</em>
</t>

приведет к

<div>
    This template was called with content:
    <em>content</em>
</div>

Пакеты активов

«умные записи» форматирование полей

Директива `` t-field`` может использоваться только при выполнении доступа к полю (`` ab``) для умной записи (результат метода `` browse``). Он может автоматически форматировать в зависимости от типа поля и интегрируется в расширенную текстовую версию веб-сайта.

`` t-options`` можно использовать для настройки полей, наиболее распространенным вариантом является `` widget``, другие параметры зависят от поля или виджета.

Отладка

`` Т-debug``

вызывает отладчик, используя `` set_trace`` API PDB. Параметр должен быть именем модуля, для которого вызывается метод `` set_trace``:

<t t-debug="pdb"/>

эквивалентно `` importlib.import_module (pdb). set_trace () ``

Помощники

Запрос на основе

Большинство QWeb на стороне Python используются в контроллерах (и во время HTTP-запросов), и в этом случае шаблоны хранятся в базе данных (как views) может быть тривиально воспроизведен путем вызова odoo.http.HttpRequest.render():

response = http.request.render('my-template', {
    'context_value': 42
})

Это автоматически создает объект Response, который может быть возвращен из контроллера (или дополнительно настроен для соответствия).

Просмотр на основе

На более глубоком уровне, чем предыдущий помощник, находится метод `` render`` в `` ir.ui.view``:

render(cr, uid, id[, values][, engine='ir.qweb][, context])

Визуализирует представление / шаблон QWeb по идентификатору базы данных или: term: external id. Шаблоны автоматически загружаются из записей `` ir.ui.view``.

Устанавливает количество значений по умолчанию в контексте рендеринга:

`` request``
текущий объект WebRequest, если есть
`` Debug``
находится ли текущий запрос (если есть) в режиме `` debug``
: FUNC: `quote_plus <werkzeug.urls.url_quote_plus> `
функция полезности кодирования URL
: Моды: json
соответствующий стандартный библиотечный модуль
: Моды: time
соответствующий стандартный библиотечный модуль
: Моды: datetime
соответствующий стандартный библиотечный модуль
relativedelta
см. модуль
`` Keep_query``
вспомогательная функция `` keep_query``
Параметры
  • values – значения контекста для передачи в QWeb для рендеринга
  • engine (str) – имя модели Odoo, используемой для рендеринга, может использоваться для локального расширения или настройки QWeb (путем создания «нового» qweb на основе `` ir.qweb`` с изменениями)

Javascript

Эксклюзивные директивы

определяющие шаблоны

Директива `` t-name`` может быть размещена только на верхнем уровне файла шаблона (прямые дочерние элементы к корню документа)

<templates>
    <t t-name="template-name">
        <!-- template code -->
    </t>
</templates>

Он не принимает никаких других параметров, но может использоваться с `` <t> Элемент или любой другой. С `` <t> `` элемент, `` <t> `` должен иметь одного ребенка.

Имя шаблона является произвольной строкой, хотя, когда несколько шаблонов связаны (например, называются под-шаблонами), обычно используются разделенные точками имена для обозначения иерархических отношений.

наследование шаблонов

Наследование шаблонов используется для изменения существующих шаблонов на месте, например, для добавления информации в шаблоны, созданные другими модулями.

Наследование шаблонов выполняется с помощью директивы t-extend`, которая принимает имя шаблона для изменения в качестве параметра.

Когда `` t-extension`` объединяется с `` t-name``, создается новый шаблон с заданным именем. В этом случае расширенный шаблон не изменяется, вместо этого директивы определяют, как создать новый шаблон.

В обоих случаях изменение затем выполняется с любым количеством подкаталогов `` t-jquery``

<t t-extend="base.template">
    <t t-jquery="ul" t-operation="append">
        <li>new element</li>
    </t>
</t>

Директивы `` t-jquery`` принимают селектор CSS. Этот селектор используется в расширенном шаблоне для выбора * узлов контекста *, к которым применяется указанная t-операция`:

`` Append``
тело узла добавляется в конец узла контекста (после последнего потомка узла контекста)
`` prepend``
тело узла добавляется к контекстному узлу (вставляется перед первым дочерним узлом контекста)
`` before``
тело узла вставляется прямо перед контекстным узлом
`` after``
тело узла вставляется сразу после контекстного узла
`` inner``
тело узла заменяет дочерние узлы контекста
`` replace``
тело узла используется для замены самого узла контекста
`` attributes``
Тело узлов должно состоять из любого количества элементов `` attribute``, каждый из которых имеет атрибут `` name`` и некоторое текстовое содержимое; для именованного атрибута узла контекста будет установлено указанное значение (либо заменяется, если оно уже существовало) или добавил если нет)
Нет операции

если `` t-операция`` не указана, тело шаблона интерпретируется как код javascript и выполняется с узлом контекста как `` this``

Предупреждение

Хотя этот режим гораздо более мощный, чем другие операции, его также гораздо сложнее отлаживать и поддерживать, но рекомендуется его избегать.

отладка

Реализация javascript QWeb предоставляет несколько отладочных хуков:

`` Т-log``

принимает параметр выражения, вычисляет выражение во время рендеринга и записывает его результат с помощью `` console.log``

<t t-set="foo" t-value="42"/>
<t t-log="foo"/>

выведет `` 42`` на консоль

`` Т-debug``

запускает точку останова отладчика во время рендеринга шаблона:

<t t-if="a_test">
    <t t-debug="">
</t>

остановит выполнение, если отладка активна (точное условие зависит от браузера и его инструментов разработки)

`` Т-js``

тело узла - это код JavaScript, выполняемый во время рендеринга шаблона. Принимает параметр `` context``, который является именем, под которым контекст рендеринга будет доступен в теле `` t-js``

<t t-set="foo" t-value="42"/>
<t t-js="ctx">
    console.log("Foo is", ctx.foo);
</t>

Помощники

core.qweb

(ядро - это модуль `` web.core``). Экземпляр: js QWeb2.Engine со всеми загруженными файлами шаблонов модулей и ссылками на стандартные вспомогательные объекты` _` (подчеркивание), `` _t`` (функция перевода) и JSON.

: ЯШ: FUNC: `core.qweb.render <QWeb2.Engine.render> `может использоваться для простой визуализации базовых шаблонов модулей

API

class QWeb2.Engine()

QWeb «рендерер» обрабатывает большую часть логики QWeb (загрузка, анализ, компиляция и рендеринг шаблонов).

Odoo Web создает экземпляр для пользователя в модуле ядра и экспортирует его в `` core.qweb``. Он также загружает все файлы шаблонов различных модулей в этот экземпляр QWeb.

A: js QWeb2.Engine также служит« пространством имен шаблона ».

QWeb2.Engine.QWeb2.Engine.render(template[, context])

Визуализирует ранее загруженный шаблон в строку, используя `` context`` (если имеется), чтобы найти переменные, к которым обращались во время рендеринга шаблона (например, строки для отображения).

Аргументы
  • template (String) – название шаблона для отображения
  • context (Object) – основное пространство имен, используемое для рендеринга шаблона
Результат
строка

Движок предоставляет другой метод, который может быть полезен в некоторых случаях (например, если вам нужно отдельное пространство имен шаблона с, в Odoo Web представления Kanban получают свой собственный экземпляр: js QWeb2.Engine, поэтому их шаблоны не сталкиваются с более общими «модульными» шаблонами):

QWeb2.Engine.QWeb2.Engine.add_template(templates)

Загружает файл шаблона (коллекцию шаблонов) в экземпляре QWeb. Шаблоны могут быть указаны как:

XML-строка
QWeb попытается проанализировать его в XML-документе, а затем загрузить его.
URL
QWeb попытается загрузить содержимое URL, а затем загрузить полученную XML-строку.
«Документ» или «Узел»
QWeb будет проходить по первому уровню документа (дочерним узлам предоставленного корня) и загружать любой именованный шаблон или переопределение шаблона.

A: js QWeb2.Engine также предоставляет различные атрибуты для настройки поведения:

QWeb2.Engine.QWeb2.Engine.prefix

Префикс используется для распознавания директив во время синтаксического анализа. Строка. По умолчанию `` t``.

QWeb2.Engine.QWeb2.Engine.debug

Логический флаг, переводящий двигатель в «режим отладки». Обычно QWeb перехватывает любую ошибку, возникающую во время выполнения шаблона. В режиме отладки он оставляет все исключения без их перехвата.

QWeb2.Engine.QWeb2.Engine.jQuery

Экземпляр jQuery, используемый во время обработки наследования шаблона. По умолчанию используется `` window.jQuery``.

QWeb2.Engine.QWeb2.Engine.preprocess_node

`` Функция``. Если присутствует, вызывается перед компиляцией каждого узла DOM в код шаблона. В Odoo Web это используется для автоматического перевода текстового содержимого и некоторых атрибутов в шаблонах. По умолчанию `` null``.
