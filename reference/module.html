манифест

Файл манифеста служит для объявления пакета Python как модуля Odoo и для указания метаданных модуля.

Это файл с именем `` __manifest __. Py`` и содержит один словарь Python, где каждый ключ определяет метаданные модуля.

{
    'name': "A Module",
    'version': '1.0',
    'depends': ['base'],
    'author': "Author Name",
    'category': 'Category',
    'description': """
    Description text
    """,
    # data files always loaded at installation
    'data': [
        'views/mymodule_view.xml',
    ],
    # data files containing optionally loaded demonstration data
    'demo': [
        'demo/demo_data.xml',
    ],
}

Доступные поля манифеста:

`` name`` (`` str``, обязательно)
удобочитаемое имя модуля
`` версия`` (`` str``)
версия этого модуля должна следовать правилам `семантического контроля версий__
`` описание`` (`` str``)
расширенное описание для модуля, в reStructuredText
`` author`` (`` str``)
имя автора модуля
`` website`` (`` str``)
URL сайта для автора модуля
`` license`` (`` str``, по умолчанию: `` LGPL-3``)

лицензия на распространение модуля. Возможные значения:

  • GPL-2
  • GPL-2 или любая более поздняя версия
  • GPL-3
  • GPL-3 или более поздняя версия
  • AGPL-3
  • LGPL-3
  • Другая лицензия, утвержденная OSI
  • OEEL-1 (лицензия Odoo Enterprise Edition v1.0)
  • OPL-1 (Собственная лицензия Odoo v1.0)
  • «Другие проприетарные»
`` category`` (`` str``, по умолчанию: `` Uncategorized``)

классификационная категория в пределах Odoo, грубая бизнес-область для модуля.

Хотя рекомендуется использовать существующие категории, поле имеет произвольную форму, и неизвестные категории создаются на лету. Иерархии категорий могут быть созданы с использованием разделителя `` / , например `` Foo / Bar, создаст категорию `` Foo``, категорию `` Bar`` в качестве дочерней категории `` Foo`` и будет установите `` Bar`` в качестве категории модуля.

`` selected`` (`` list (str) ``)

Модули Odoo, которые должны быть загружены до этого, либо потому, что этот модуль использует функции, которые они создают, либо потому, что он изменяет ресурсы, которые они определяют.

Когда модуль установлен, все его зависимости устанавливаются перед ним. Аналогично, зависимости загружаются до загрузки модуля.

`` data`` (`` list (str) ``)
Список файлов данных, которые всегда должны быть установлены или обновлены вместе с модулем. Список путей из корневого каталога модуля
`` demo`` (`` list (str) ``)
Список файлов данных, которые устанавливаются или обновляются только в * демонстрационном режиме *
`` auto_install`` (`` bool``, по умолчанию: `` False``)

Если `` True``, этот модуль будет автоматически установлен, если установлены все его зависимости.

Обычно используется для «модулей связи», реализующих синергетическую интеграцию между двумя независимыми модулями.

Например, `` sale_crm`` зависит от `` sale`` и `` crm`` и имеет значение `` auto_install``. Когда установлены `` sale`` и `` crm``, он автоматически добавляет отслеживание CRM-кампаний в заказы на продажу без того, чтобы `` sale`` или `` crm`` знали друг о друге

`` external_dependencies`` (`` dict (key = list (str)) ``)

Словарь, содержащий питон и / или бинарные зависимости.

Для зависимостей python ключ `` python`` должен быть определен для этого словаря, и ему должен быть присвоен список импортируемых модулей python.

Для двоичных зависимостей ключ `` bin`` должен быть определен для этого словаря, и ему должен быть назначен список имен исполняемых двоичных файлов.

Модуль не будет установлен, если модуль Python не установлен на хост-машине или двоичный исполняемый файл не найден в переменной окружения PATH хост-машины.

`` application`` (`` bool``, по умолчанию: `` False``)
Должен ли модуль рассматриваться как полноценное приложение (`` True``) или просто как технический модуль (`` False``), который предоставляет некоторую дополнительную функциональность существующему модулю приложения.
`` css`` (`` list (str) ``)
Укажите файлы css с настраиваемыми правилами для импорта, эти файлы должны находиться в `` static / src / css`` внутри модуля.
`` images`` (`` list (str) ``)
Укажите файлы изображений, которые будут использоваться модулем.
`` installable`` (`` bool`` default: `` True``)
Должен ли пользователь иметь возможность установить модуль из веб-интерфейса или нет.
`` сопровождающий`` (`` str``)
Физическое или юридическое лицо, отвечающее за обслуживание этого модуля, по умолчанию предполагается, что автор является сопровождающим.
`` {pre_init, post_init, uninstall} _hook`` (`` str``)

Хуки для установки / удаления модуля, их значением должна быть строка, представляющая имя функции, определенной внутри модуля `` __init __. Py``.

`` pre_init_hook`` принимает курсор в качестве единственного аргумента, эта функция выполняется перед установкой модуля.

`` post_init_hook`` принимает курсор и реестр в качестве аргументов, эта функция выполняется сразу после установки модуля.

`` uninstall_hook`` принимает курсор и реестр в качестве аргументов, эта функция выполняется после удаления модуля.

Эти хуки должны использоваться только тогда, когда настройка / очистка, необходимая для этого модуля, либо чрезвычайно сложна, либо невозможна через API.

active (bool)
This indicates whether this module is to be used or not. It is generally used to archive obsolete modules.
