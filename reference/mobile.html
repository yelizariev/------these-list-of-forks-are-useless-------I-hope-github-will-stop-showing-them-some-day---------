Вступление

В Odoo 10.0 мы выпустили мобильное приложение, которое позволяет вам получить доступ ко всем ** приложениям Odoo ** (даже к вашим настроенным модулям).

Приложение представляет собой комбинацию ** Odoo Web ** и ** Native Mobile компонентов **. Другими словами, это экземпляр Odoo Web, загруженный внутри собственного мобильного контейнера WebView.

На этой странице описано, как получить доступ к собственным мобильным компонентам, таким как Камера, Вибрация, Уведомления и Тост, через Odoo Web (через JavaScript). Для этого вам не нужно быть разработчиком мобильных приложений. Если вы знакомы с Odoo JavaScript API, вы можете получить доступ ко всем доступным мобильным функциям.

Предупреждение

Эти функции работают только с ** Odoo Enterprise 10.0 + **

Как это работает?

Внутренние разработки мобильного приложения:

Конечно, это веб-страница, которая загружается в контейнер Mobile Native Web. Но он интегрирован таким образом, что вы можете получить доступ к собственным ресурсам из вашего веб-JavaScript.

Веб-страницы (Odoo Web) находятся в верхней части каждого уровня, где второй уровень представляет собой мост между Odoo Web (JS) и собственными мобильными компонентами.

Когда любой вызов из JavaScript запускается, он проходит через Bridge, а Bridge передает его нативному вызывающему для выполнения этого действия.

Когда нативный компонент выполнил свою работу, он снова передается в Bridge, и вы получаете вывод в JavaScript.

Время обработки, занимаемое компонентом Native, зависит от того, что вы запрашиваете у ресурсов Native. Например, камера или местоположение GPS.

Как это использовать?

Как и в случае с Odoo Web Framework, Mobile API можно использовать где угодно, получая объект от ** web_mobile.rpc **

Мобильный объект RPC предоставляет список доступных методов (это работает только с мобильным приложением).

Проверьте, доступен ли метод, а затем выполните его.

методы

Примечание

Каждый из методов возвращает объект JQuery Deffered, который возвращает словарь данных JSON

Показать тост в устройстве

showToast()
Аргументы
  • args (object) – ** сообщение ** текст для отображения

Тост обеспечивает простой отзыв об операции в небольшом всплывающем окне. Он только заполняет объем, необходимый для сообщения, и текущая активность остается видимой и интерактивной.

mobile.methods.showToast({'message': 'Message sent'});

Вибрационное устройство

vibrate()
Аргументы
  • args (object) – Постоянно вибрирует в течение указанного периода времени (в миллисекундах).

Вибрировать мобильное устройство с заданной продолжительностью.

mobile.methods.vibrate({'duration': 100});

Показать закусочную с действием

showSnackBar()
Аргументы
  • args (object) – (* обязательно ) * Сообщение ** для отображения в панели закусок и действие ** метка кнопки ** в панели закусок (необязательно)
Результат
`` True``, если пользователь нажимает кнопку «Действие», `` False``, если SnackBar автоматически отключается через некоторое время.

Закусочные обеспечивают легкую обратную связь об операции. Они показывают краткое сообщение в нижней части экрана на мобильном устройстве или в нижнем левом углу на более крупных устройствах. Закусочные панели отображаются над всеми остальными элементами на экране, и одновременно может отображаться только один элемент.

mobile.methods.showSnackBar({'message': 'Message is deleted', 'btn_text': 'Undo'}).then(function(result){
        if(result){
                // Do undo operation
        }else{
                // Snack Bar dismissed
        }
});

Отображение уведомления

showNotification()
Аргументы
  • args (object) – ** заголовок ** (первая строка) уведомления, ** сообщение ** (вторая строка) уведомления, в стандартном уведомлении.

Уведомление - это сообщение, которое вы можете отобразить пользователю за пределами обычного пользовательского интерфейса вашего приложения. Когда вы сообщаете системе о выдаче уведомления, оно сначала отображается в виде значка в области уведомлений. Чтобы просмотреть подробности уведомления, пользователь открывает ящик уведомлений. Как область уведомлений, так и область уведомлений являются управляемыми системой областями, которые пользователь может просматривать в любое время.

mobile.showNotification({'title': 'Simple Notification', 'message': 'This is a test for a simple notification'})

Создать контакт в устройстве

addContact()
Аргументы
  • args (object) – Словарь с контактными данными. Возможные ключи (имя, мобильный, телефон, факс, электронная почта, веб-сайт, улица, улица2, country_id, state_id, город, почтовый индекс, parent_id, функция и изображение)

Создайте новый контакт устройства с указанными контактными данными.

var contact = {
        'name': 'Michel Fletcher',
        'mobile': '9999999999',
        'phone': '7954856587',
        'fax': '765898745',
        'email': 'michel.fletcher@agrolait.example.com',
        'website': 'http://www.agrolait.com',
        'street': '69 rue de Namur',
        'street2': false,
        'country_id': [21, 'Belgium'],
        'state_id': false,
        'city': 'Wavre',
        'zip': '1300',
        'parent_id': [8, 'Agrolait'],
        'function': 'Analyst',
        'image': '<<BASE 64 Image Data>>'
}

mobile.methods.addContact(contact);

Сканирование штрих-кодов

scanBarcode()
Результат
Сканировал `` код`` с любого штрих-кода

Штрих-код API обнаруживает штрих-коды в режиме реального времени, на устройстве, в любой ориентации.

Штрих-код API может считывать следующие форматы штрих-кода:

  • Штрих-коды 1D: EAN-13, EAN-8, UPC-A, UPC-E, Код-39, Код-93, Код-128, ITF, Codabar
  • 2D штрих-коды: QR-код, матрица данных, PDF-417, AZTEC
mobile.methods.scanBarcode().then(function(code){
        if(code){
                // Perform operation with the scanned code
        }
});

switchAccount()

Используйте switchAccount для переключения с одной учетной записи на другую на устройстве.

mobile.methods.switchAccount();
