Edit on GitHub

Отчеты пишутся в HTML / QWeb, как и все обычные представления в Odoo. Вы можете использовать обычные QWeb control flow tools. Сам рендеринг PDF выполняется wkhtmltopdf.

Если вы хотите создать отчет по определенной модели, вам нужно определить это отчет и отчет, который он будет использовать. При желании вы также можете указать конкретный отчет для этого отчета. Наконец, если вам нужен доступ не только к вашей модели, вы можете определить класс отчет, который предоставит вам доступ к большему количеству моделей и записей в шаблоне.

отчет

Каждый отчет должен быть объявлен действием report action.

Для простоты ярлык `` <report> Элемент `` доступен для определения отчета, а не для настройки the action <reference/actions/report>` <report> `` может принимать следующие атрибуты:

`` Id``
сгенерированная запись: term: external id
`` имя`` (обязательно)
полезно только как мнемоника / описание отчета при поиске в списке какого-либо типа
`` модель`` (обязательно)
модель вашего отчета
`` report_type`` (обязательно)
или `` qweb-pdf`` для отчетов в формате PDF или `` qweb-html`` для HTML
`` report_name``
название вашего отчета (который будет именем вывода PDF)
`` groups``
Many2many field to the groups allowed to view/use the current report
`` attachment_use``
если установлено значение True, отчет будет сохранен как вложение записи, используя имя, сгенерированное выражением `` attachment``; Вы можете использовать это, если вам нужно, чтобы ваш отчет был сгенерирован только один раз (например, по юридическим причинам)
`` attachment``
выражение python, определяющее имя отчета; запись доступна как переменная `` объект``
`` Paperformat``
внешний идентификатор формата бумаги, который вы хотите использовать (по умолчанию используется формат бумаги компании, если он не указан)

Пример:

<report
    id="account_invoices"
    model="account.invoice"
    string="Invoices"
    report_type="qweb-pdf"
    name="account.report_invoice"
    file="account.report_invoice"
    attachment_use="True"
    attachment="(object.state in ('open','paid')) and
        ('INV'+(object.number or '').replace('/','')+'.pdf')"
/>

Шаблон отчета

Минимальный жизнеспособный шаблон

Минимальный шаблон будет выглядеть так:

<template id="report_invoice">
    <t t-call="web.html_container">
        <t t-foreach="docs" t-as="o">
            <t t-call="web.external_layout">
                <div class="page">
                    <h2>Report title</h2>
                    <p>This object's name is <span t-field="o.name"/></p>
                </div>
            </t>
        </t>
    </t>
</template>

Вызов `` external_layout`` добавит заголовок и нижний колонтитул по умолчанию в ваш отчет. Тело PDF будет содержимым внутри `` <div class=»page»> . `` Id шаблона должен быть именем, указанным в объявлении отчета; например, `` account.report_invoice`` для вышеприведенного отчета. Поскольку это шаблон QWeb, вы можете получить доступ ко всем полям объектов `` docs` ', полученных шаблоном.

Есть несколько конкретных переменных, доступных в отчетах, в основном:

`` docs``
записи для текущего отчета
`` doc_ids``
список идентификаторов для записей `` docs``
`` doc_model``
модель для записи `` документов``
`` Time``
ссылка на: mod: python: time из стандартной библиотеки Python
`` user``
`` res.user`` запись для пользователя, печатающего отчет
`` res_company``
запись для текущей `` пользователя`` компании

Если вы хотите получить доступ к другим записям / моделям в шаблоне, вам потребуется a custom report.

Переводимые шаблоны

Если вы хотите перевести отчеты (например, на язык партнера), вам нужно определить два шаблона:

  • Основной шаблон отчета
  • Переводимый документ

Затем вы можете вызвать переводимый документ из вашего основного шаблона с атрибутом `` t-lang``, для которого установлен код языка (например, `` fr`` или `` en_US``) или в поле записи. Вам также нужно будет повторно просмотреть связанные записи в соответствующем контексте, если вы используете переводимые поля (например, названия стран, условия продаж и т. Д.)

Предупреждение

Если в вашем шаблоне отчета не используются переводимые поля записей, повторный просмотр записи на другом языке * не * необходим и повлияет на производительность.

Например, давайте посмотрим на отчет «Заказ на продажу» из модуля «Продажа»:

<!-- Main template -->
<template id="report_saleorder">
    <t t-call="web.html_container">
        <t t-foreach="docs" t-as="doc">
            <t t-call="sale.report_saleorder_document" t-lang="doc.partner_id.lang"/>
        </t>
    </t>
</template>

<!-- Translatable template -->
<template id="report_saleorder_document">
    <!-- Re-browse of the record with the partner lang -->
    <t t-set="doc" t-value="doc.with_context(lang=doc.partner_id.lang)" />
    <t t-call="web.external_layout">
        <div class="page">
            <div class="oe_structure"/>
            <div class="row">
                <div class="col-6">
                    <strong t-if="doc.partner_shipping_id == doc.partner_invoice_id">Invoice and shipping address:</strong>
                    <strong t-if="doc.partner_shipping_id != doc.partner_invoice_id">Invoice address:</strong>
                    <div t-field="doc.partner_invoice_id" t-options="{&quot;no_marker&quot;: True}"/>
                <...>
            <div class="oe_structure"/>
        </div>
    </t>
</template>

Основной шаблон вызывает переводимый шаблон с `` doc.partner_id.lang`` в качестве параметра `` t-lang``, поэтому он будет отображаться на языке партнера. Таким образом, каждый Заказ на продажу будет напечатан на языке соответствующего покупателя. Если вы хотите перевести только тело документа, но сохранить верхний и нижний колонтитулы на языке по умолчанию, вы можете вызвать внешний макет отчета следующим образом:

<t t-call="web.external_layout" t-lang="en_US">

Совет

Обратите внимание, что это работает только при вызове внешних шаблонов, вы не сможете перевести часть документа, установив атрибут `` t-lang`` на узле xml, отличном от `` t-call``. Если вы хотите перевести часть шаблона, вы можете создать внешний шаблон с этим частичным шаблоном и вызвать его из основного с атрибутом `` t-lang``.

штрихкоды

Штрих-коды - это изображения, возвращаемые контроллером, и они могут быть легко встроены в отчеты благодаря синтаксису QWeb (например, см . атрибуты):

<img t-att-src="'/report/barcode/QR/%s' % 'My text in qr code'"/>

Дополнительные параметры могут быть переданы в виде строки запроса

<img t-att-src="'/report/barcode/?
    type=%s&value=%s&width=%s&height=%s'%('QR', 'text', 200, 200)"/>

Полезные замечания

  • Классы Twitter Bootstrap и FontAwesome можно использовать в шаблоне отчета
  • Локальный CSS можно поставить прямо в шаблон

  • Глобальный CSS можно вставить в основной макет отчета, унаследовав его шаблон и вставив свой CSS

    <template id="report_saleorder_style" inherit_id="report.style">
  <xpath expr=".">
    <t>
      .example-css-class {
        background-color: red;
      }
    </t>
  </xpath>
</template>
  • Если кажется, что в вашем отчете PDF отсутствуют стили, проверьте these instructions.

Формат бумаги

Бумажные форматы являются записями `` report.paperformat`` и могут содержать следующие атрибуты:

`` имя`` (обязательно)
полезно только как мнемоника / описание отчета при поиске в списке какого-либо типа
`` Description``
небольшое описание вашего формата
`` format``
либо предопределенный формат (от A0 до A9, от B0 до B10, Legal, Letter, Tabloid, …) или `` custom``; A4 по умолчанию. Нельзя использовать нестандартный формат, если вы определяете размеры страницы.
`` Dpi``
выходной DPI; 90 по умолчанию
`` margin_top``, `` margin_bottom``, `` margin_left``, `` margin_right``
размеры полей в мм
`` page_height``, `` page_width``
размеры страницы в мм
`` Orientation``
Пейзаж или Портрет
`` Header_line``
логическое значение для отображения строки заголовка
`` Header_spacing``
расстояние между жатками в мм

Пример:

<record id="paperformat_frenchcheck" model="report.paperformat">
    <field name="name">French Bank Check</field>
    <field name="default" eval="True"/>
    <field name="format">custom</field>
    <field name="page_height">80</field>
    <field name="page_width">175</field>
    <field name="orientation">Portrait</field>
    <field name="margin_top">3</field>
    <field name="margin_bottom">3</field>
    <field name="margin_left">3</field>
    <field name="margin_right">3</field>
    <field name="header_line" eval="False"/>
    <field name="header_spacing">3</field>
    <field name="dpi">80</field>
</record>

Пользовательские отчеты

Модель отчета имеет функцию `` get_html`` по умолчанию, которая ищет модель с именем: samp: report. {Module.report_name}. Если он существует, он будет использовать его для вызова механизма QWeb; в противном случае будет использоваться универсальная функция. Если вы хотите настроить отчеты, включив в шаблон больше элементов (например, записи других моделей), вы можете определить эту модель, перезаписать функцию `` _get_report_values`` и передать объекты в словарь `` docargs`` :

from odoo import api, models

class ParticularReport(models.AbstractModel):
    _name = 'report.module.report_name'

    @api.model
    def _get_report_values(self, docids, data=None):
        report_obj = self.env['ir.actions.report']
        report = report_obj._get_report_from_name('module.report_name')
        docargs = {
            'doc_ids': docids,
            'doc_model': report.model,
            'docs': self,
        }
        return docargs

Пользовательские шрифты

Если вы хотите использовать пользовательские шрифты, вам нужно добавить свой собственный шрифт и связанный с ним less / CSS в пакет ресурсов `` web.reports_assets_common``. Добавление пользовательских шрифтов в `` web.assets_common`` или `` web.assets_backend`` не сделает ваш шрифт доступным в отчетах QWeb.

Пример:

<template id="report_assets_common_custom_fonts" name="Custom QWeb fonts" inherit_id="web.report_assets_common">
    <xpath expr="." position="inside">
        <link href="/your_module/static/src/less/fonts.less" rel="stylesheet" type="text/less"/>
    </xpath>
</template>

Вам нужно будет определить ваш `` @ font-face`` в этом меньшем файле, даже если вы использовали его в другом пакете ресурсов (кроме `` web.reports_assets_common``).

Пример:

@font-face {
    font-family: 'MonixBold';
    src: local('MonixBold'), local('MonixBold'), url(/your_module/static/src/fonts/MonixBold-Regular.otf) format('opentype');
}

.h1-title-big {
    font-family: MonixBold;
    font-size: 60px;
    color: #3399cc;
}

После того, как вы добавили меньшее в свой пакет ресурсов, вы можете использовать классы - в этом примере `` h1-title-big`` - в своем пользовательском отчете QWeb.

Отчеты являются веб-страницами

Отчеты генерируются динамически модулем отчетов и могут быть доступны напрямую через URL:

Например, вы можете получить доступ к отчету о заказах на продажу в html-режиме, перейдя в http: // <server-address> /report/html/sale.report_saleorder/38

Или вы можете получить доступ к PDF-версии по адресу http: // <server-address> /report/pdf/sale.report_saleorder/38
