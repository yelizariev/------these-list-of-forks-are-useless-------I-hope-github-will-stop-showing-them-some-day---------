Существует несколько способов установить Odoo или не устанавливать его вообще, в зависимости от предполагаемого варианта использования.

Этот документ пытается описать большинство вариантов установки.

: Ссылка: Настройка / установка / online
Самый простой способ использовать Odoo в производстве или попробовать его.
: Ссылка: Настройка / установка / packaged
Подходит для тестирования Odoo, разработки модулей и может использоваться для долгосрочного использования в производстве с дополнительными работами по развертыванию и обслуживанию.
: Ссылка: Настройка / установка / source
Обеспечивает большую гибкость: например, позволяет запускать несколько версий Odoo в одной системе. Хорошо подходит для разработки модулей, может использоваться в качестве основы для развертывания производства.
: Ссылка: Настройка / установка / docker
Если вы обычно используете docker для разработки или развертывания, официальный базовый образ docker доступен.

издания

Существует две разные версии Odoo: версии для сообщества и для предприятий. Использование корпоративной версии возможно на нашем SaaS, а доступ к коду ограничен для корпоративных клиентов и партнеров. Версия сообщества доступна всем.

Если вы уже используете версию Community и хотите обновить ее до Enterprise, обратитесь к От сообщества к предприятию (за исключением От сообщества к предприятию).

В сети

демонстрация

Чтобы просто получить краткое представление об Odoo, доступны примеры demo. Это общие экземпляры, которые живут всего несколько часов и могут быть использованы для просмотра и тестирования без каких-либо обязательств.

Demo экземпляры не требуют локальной установки, только веб-браузер.

SaaS

Начав с простого, полностью управляемого и перенесенного Odoo SA, SaaS в Odoo предоставляет частные экземпляры и запускается бесплатно. Его можно использовать для обнаружения и тестирования Odoo и выполнения настроек, не связанных с кодом (т. Е. Несовместимых с пользовательскими модулями или магазином приложений Odoo), без необходимости его локальной установки.

Может использоваться как для тестирования Odoo, так и для долгосрочного использования.

Как и экземпляры demo, экземпляры SaaS не требуют локальной установки, достаточно веб-браузера.

Упакованные установщики

Odoo предоставляет упакованные установщики для Windows, дистрибутивов на основе deb (Debian, Ubuntu, …) и дистрибутивов на основе RPM (Fedora, CentOS, RHEL, …) для версий Community и Enterprise.

Эти пакеты автоматически устанавливают все зависимости (для версии Community), но их сложно поддерживать в актуальном состоянии.

Официальные пакеты Сообщества со всеми соответствующими требованиями зависимости доступны на нашем nightly сервере. Пакеты Communtiy и Enterprise можно загрузить с нашей страницы download (для загрузки пакетов Enterprise необходимо войти в систему как платящий клиент или партнер).

Windows

  • Загрузите установщик с нашего ночного сервера (только для сообщества) или установщика Windows со страницы загрузки (любая версия)

  • Запустите загруженный файл

    Предупреждение

    в Windows 8 может появиться предупреждение под названием «Windows защитил ваш компьютер». Нажмите: guilabel: More Info затем: guilabel:` Run anyway`

  • Принять приглашение UAC
  • Пройдите различные этапы установки

Odoo будет автоматически запущен в конце установки.

Linux

Debian / Ubuntu

Пакет deb для Odoo 13.0 в настоящее время поддерживает Debian Buster,` Ubuntu 18.04`_ или выше.

Подготовить

Для правильной работы Odoo необходим сервер PostgreSQL. Конфигурация по умолчанию для пакета Odoo 'deb' заключается в использовании сервера PostgreSQL на том же хосте, что и ваш экземпляр Odoo. Выполните следующую команду от имени пользователя root для установки сервера PostgreSQL:

# apt-get install postgresql -y

Чтобы распечатать отчеты в формате PDF, вы должны установить wkhtmltopdf самостоятельно: версия wkhtmltopdf, доступная в репозиториях Debian, не поддерживает верхние и нижние колонтитулы, поэтому она не используется в качестве прямой зависимости. Рекомендуемая версия - 0.12.5. Она доступна на странице загрузки wkhtmltopdf в разделе архива. Ранее рекомендованная версия 0.12.1 является хорошей альтернативой. Более подробную информацию о различных версиях и их причудах можно найти в нашей wiki.

вместилище

Odoo SA предоставляет репозиторий, который можно использовать с дистрибутивами Debian и Ubuntu. Его можно использовать для установки Odoo Community Edition, выполнив следующие команды от имени root:

# wget -O - https://nightly.odoo.com/odoo.key | apt-key add -
# echo "deb http://nightly.odoo.com/13.0/nightly/deb/ ./" >> /etc/apt/sources.list.d/odoo.list
# apt-get update && apt-get install odoo

Затем вы можете использовать обычную команду `` apt-get upgrade``, чтобы поддерживать вашу установку в актуальном состоянии.

На данный момент репозитория для Enterprise Edition нет.

Деб Пакет

Вместо использования репозитория, как описано выше, пакет 'deb' можно скачать здесь:

  • Community Edition: nightly
  • Enterprise Edition Скачать

Затем вы можете использовать `` gdebi``:

# gdebi <path_to_installation_package>

Или `` dpkg``:

# dpkg -i <path_to_installation_package> # this probably fails with missing dependencies
# apt-get install -f # should install the missing dependencies
# dpkg -i <path_to_installation_package>

Это установит Odoo как сервис, создаст нужного пользователя PostgreSQL и автоматически запустит сервер.

Предупреждение

Пакет Debian python3-xlwt не существует ни в Debian Buster, ни в Ubuntu 18.04. Этот модуль python необходим для экспорта в формат xls.

Если вам нужна эта функция, вы можете установить ее вручную. Один из способов сделать это - просто использовать pip3:

$ sudo pip3 install xlwt

Предупреждение

Debian 9 и Ubuntu не предоставляют пакет для модуля Python num2words. Текстовые суммы не будут отображаться Odoo, и это может вызвать проблемы с модулем l10n_mx_edi.

Если вам нужна эта функция, вы можете установить модуль python следующим образом:

$ sudo pip3 install num2words

мягкая фетровая шляпа

Пакет Odoo 13.0 'rpm' поддерживает Fedora 30.

Подготовить

Для правильной работы Odoo необходим сервер PostgreSQL. Предполагая, что команда 'sudo' доступна и настроена правильно, выполните следующие команды:

$ sudo dnf install -y postgresql-server
$ sudo postgresql-setup --initdb --unit postgresql
$ sudo systemctl enable postgresql
$ sudo systemctl start postgresql

Чтобы распечатать отчеты в формате PDF, вы должны установить wkhtmltopdf самостоятельно: версия wkhtmltopdf, доступная в репозиториях Debian, не поддерживает верхние и нижние колонтитулы, поэтому она не используется в качестве прямой зависимости. Рекомендуемая версия - 0.12.5. Она доступна на странице загрузки wkhtmltopdf в разделе архива. Ранее рекомендованная версия 0.12.1 является хорошей альтернативой. Более подробную информацию о различных версиях и их причудах можно найти в нашей wiki.

вместилище

Odoo SA предоставляет репозиторий, который можно использовать с дистрибутивами Fedora. Его можно использовать для установки Odoo Community Edition, выполнив следующие команды:

$ sudo dnf config-manager --add-repo=https://nightly.odoo.com/13.0/nightly/rpm/odoo.repo
$ sudo dnf install -y odoo
$ sudo systemctl enable odoo
$ sudo systemctl start odoo
RPM пакет

Вместо использования репозитория, как описано выше, пакет 'rpm' можно скачать здесь:

  • Community Edition: nightly
  • Enterprise Edition Скачать

После загрузки пакет можно установить с помощью диспетчера пакетов 'dnf':

$ sudo dnf localinstall odoo_13.0.latest.noarch.rpm
$ sudo systemctl enable odoo
$ sudo systemctl start odoo

Источник Установить

«Установка» в исходном коде на самом деле заключается в том, чтобы не устанавливать Odoo, а вместо этого запускать его непосредственно из источника.

Это может быть более удобным для разработчиков модулей, поскольку исходный код Odoo более доступен, чем при использовании пакетной установки (для информации или для создания этой документации и ее доступности в автономном режиме).

Это также делает запуск и остановку Odoo более гибким и явным, чем сервисы, установленные пакетными установками, и позволяет переопределять настройки с помощью :ref:`command-line parameters <reference/cmdline>`без необходимости редактировать файл конфигурации.

Наконец, он обеспечивает больший контроль над настройками системы и позволяет легче поддерживать (и запускать) несколько версий Odoo бок о бок.

Windows

Получить источники

Есть два способа получить исходный код Odoo: в виде zip ** архива ** или через ** git **.

Архив

Community Edition:

  • Официальная страница загрузки
  • GitHub репозиторий
  • Ночной сервер

Enterprise Edition:

  • Официальная страница загрузки
  • GitHub репозиторий
Гит

Следующее требует, чтобы git был установлен на вашем компьютере и имел базовые знания о командах git.

Community Edition:

C:\> git clone https://github.com/odoo/odoo.git

Enterprise Edition: (см.: Ref: setup / install / editions для получения доступа)

C:\> git clone https://github.com/odoo/enterprise.git

Примечание

** Репозиторий Enterprise git не содержит полного исходного кода Odoo **. Это всего лишь набор дополнительных дополнений. Основной код сервера находится в версии сообщества. Запуск версии Enterprise фактически означает запуск сервера из версии Community с опцией addons-path, установленной в папку с версией Enterprise. Вам нужно клонировать репозиторий Community и Enterprise, чтобы иметь работающую установку Odoo Enterprise.

Подготовить

питон

Для запуска Odoo требуется Python 3.6 или новее. Используйте официальный установщик Python 3 ' для загрузки и установки Python 3 на вашем компьютере.

Во время установки установите флажок ** Добавить Python 3 в PATH **, затем нажмите ** Настройка установки ** и убедитесь, что установлен флажок ** pip **.

Примечание

Если Python 3 уже установлен, убедитесь, что он версии 3.6 или выше, так как предыдущие версии не совместимы с Odoo.

C:\> python3 --version

Убедитесь также, что pip установлен для этой версии.

C:\> pip3 --version
PostgreSQL

Odoo использует PostgreSQL в качестве системы управления базами данных. Загрузите и установите последнюю версию PostgreSQL.

По умолчанию единственным пользователем является postgres, но Odoo запрещает подключение как` postgres`, поэтому вам нужно создать нового пользователя PostgreSQL:

  1. Добавить каталог bin PostgreSQL (по умолчанию: C: \ Program Files \ PostgreSQL \ <version> \ bin`) к вашему PATH.

  2. Создайте пользователя postgres с паролем с помощью интерфейса администратора pg:

    • Откройте ** pgAdminIII **.
    • Дважды щелкните сервер, чтобы создать соединение.
    • Выберите Edit ‣ New Object ‣ New Login Role.
    • Введите имя пользователя в поле ** Role Name ** (например, odoo).
    • Откройте вкладку ** Definition ** и введите пароль (например, `` odoo``), затем нажмите ** OK **.
зависимости

Зависимости Odoo перечислены в файле needs.txt, расположенном в корне каталога сообщества Odoo. Большинство из них можно установить с помощью ** pip **.

Совет

Может быть предпочтительнее не смешивать пакеты модулей Python между различными экземплярами Odoo или с вашей системой. Вы можете использовать virtualenv для создания изолированных сред Python.

Перейдите к пути установки вашего сообщества Odoo (YourOdooCommunityPath) и запустите ** pip ** в файле требований:

C:\> cd \YourOdooCommunityPath
C:\YourOdooCommunityPath> C:\Python35\Scripts\pip.exe install -r requirements.txt

Предупреждение

Некоторые зависимости не могут быть установлены через pip и требуют установки вручную. Особенно:

  • psycopg должен быть установлен с` этим установщиком <http://www.stickpeople.com/projects/python/win-psycopg/>`_.
  • wkhtmltopdf должен быть установлен в версии` 0.12.5 <the wkhtmltopdf download page>`_ для поддержки верхних и нижних колонтитулов. Смотрите нашу вики <https://github.com/odoo/odoo/wiki/Wkhtmltopdf>`_ для более подробной информации о различных версиях.

Для языков с интерфейсом справа налево (таких как арабский или иврит) необходим пакет rtlcss:

  1. Загрузите и установите nodejs.

  2. Установите rtlcss:

    C:\> npm install -g rtlcss
  3. Отредактируйте переменную системного окружения PATH, чтобы добавить папку, в которой находится` rtlcss.cmd` (обычно: `C: \ Users \ <user> \ AppData \ Roaming \ НПМ \ `).

Бег Оду

После того, как все зависимости установлены, Odoo может быть запущен с помощью odoo-bin, интерфейса командной строки сервера. Он находится в корне каталога сообщества Odoo.

Чтобы настроить сервер, вы можете указать command-line arguments.

Совет

Для редакции Enterprise вы должны добавить путь к аддонам enterprise к аргументу` addons-path`. Обратите внимание, что он должен идти перед другими путями в addons-path для правильной загрузки аддонов.

Общие необходимые конфигурации:

  • PostgreSQL пользователь и пароль.
  • Пользовательские пути аддонов за пределами значений по умолчанию, чтобы загрузить свои собственные модули.

Типичный способ запустить сервер:

C:\YourOdooCommunityPath> python3 odoo-bin -r dbuser -w dbpassword --addons-path=addons,../mymodules --db-filter=mydb$

Где YourOdooCommunityPath - это путь установки сообщества Odoo,` dbuser` - это логин PostgreSQL, dbpassword - пароль PostgreSQL,` ../ mymodules` - каталог с дополнительными дополнениями, а mydb - база данных по умолчанию для служить на localhost: 8069.

Подготовить

питон

Для запуска Odoo требуется Python 3.6 или новее. Используйте ваш менеджер пакетов для загрузки и установки Python 3 на вашем компьютере, если это еще не сделано.

Примечание

Если Python 3 уже установлен, убедитесь, что он версии 3.6 или выше, так как предыдущие версии не совместимы с Odoo.

$ python3 --version

Убедитесь также, что pip установлен для этой версии.

$ pip3 --version
PostgreSQL

Odoo использует PostgreSQL в качестве системы управления базами данных. Используйте менеджер пакетов для загрузки и установки последней версии PostgreSQL.

По умолчанию единственным пользователем является postgres, но Odoo запрещает подключение как` postgres`, поэтому вам нужно создать нового пользователя PostgreSQL:

$ sudo -u postgres createuser -s $USER
$ createdb $USER

Примечание

Поскольку ваш пользователь PostgreSQL имеет то же имя, что и логин Unix, вы сможете подключиться к базе данных без пароля.

зависимости

Зависимости Odoo перечислены в файле needs.txt, расположенном в корне каталога сообщества Odoo. Большинство из них можно установить с помощью ** pip **, хотя некоторые библиотеки требуют установки другого системного пакета.

Совет

Может быть предпочтительнее не смешивать пакеты модулей Python между различными экземплярами Odoo или с вашей системой. Вы можете использовать virtualenv для создания изолированных сред Python.

В Debian / Ubuntu файл требований требует установки следующих пакетов:

# apt install libpq-dev libldap2-dev libsasl2-dev libxslt1-dev
# apt install python3-setuptools python3-wheel

Некоторые установки требуют, чтобы колесо также устанавливалось через пипс:

$ pip3 install wheel

Перейдите к пути установки вашего сообщества Odoo (YourOdooCommunityPath) и запустите ** pip ** в файле требований:

$ cd /YourOdooCommunityPath
/YourOdooCommunityPath$ pip3 install -r requirements.txt

Предупреждение

Для библиотек, использующих нативный код (Pillow, lxml, greenlet, gevent, psycopg2, ldap), может потребоваться установить инструменты разработки и нативные зависимости, прежде чем pip сможет установить сами зависимости. Они доступны в пакетах -dev или` -devel` для Python, PostgreSQL, libxml2, libxslt1, libevent, libsasl2 и libldap2.

Предупреждение

Некоторые зависимости не могут быть установлены через pip и требуют установки вручную. Особенно:

  • wkhtmltopdf должен быть установлен в версии` 0.12.5 <the wkhtmltopdf download page>`_ для поддержки верхних и нижних колонтитулов. Смотрите нашу вики <https://github.com/odoo/odoo/wiki/Wkhtmltopdf>`_ для более подробной информации о различных версиях.

Для языков с интерфейсом справа налево (таких как арабский или иврит) необходим пакет rtlcss:

  1. Загрузите и установите ** nodejs ** и ** npm ** с вашим менеджером пакетов.

  2. Установите rtlcss:

    $ sudo npm install -g rtlcss

Бег Оду

После того, как все зависимости установлены, Odoo может быть запущен с помощью odoo-bin, интерфейса командной строки сервера. Он находится в корне каталога сообщества Odoo.

Чтобы настроить сервер, вы можете указать command-line arguments.

Совет

Для редакции Enterprise вы должны добавить путь к аддонам enterprise к аргументу` addons-path`. Обратите внимание, что он должен идти перед другими путями в addons-path для правильной загрузки аддонов.

Общие необходимые конфигурации:

  • PostgreSQL пользователь и пароль. У Odoo нет значений по умолчанию, кроме значений по умолчанию для psycopg2 <http://initd.org/psycopg/docs/module.html>`_: соединяется через сокет UNIX на порту` 5432` с текущим пользователем и без пароля.
  • Пользовательские пути аддонов за пределами значений по умолчанию, чтобы загрузить свои собственные модули.

Типичный способ запустить сервер:

/YourOdooCommunityPath$ python3 odoo-bin --addons-path=addons,../mymodules --db-filter=mydb$

Где YourOdooCommunityPath - это путь установки Сообщества Odoo,` ../ mymodules` - это каталог с дополнительными надстройками, а mydb - база данных по умолчанию для обслуживания на` localhost: 8069`.

Получить источники

Есть два способа получить исходный код Odoo: в виде zip ** архива ** или через ** git **.

Архив

Community Edition:

  • Официальная страница загрузки
  • GitHub репозиторий
  • Ночной сервер

Enterprise Edition:

  • Официальная страница загрузки
  • GitHub репозиторий
Гит

Следующее требует, чтобы git был установлен на вашем компьютере и имел базовые знания о командах git.

Community Edition:

$ git clone https://github.com/odoo/odoo.git

Enterprise Edition: (см.: Ref: setup / install / editions для получения доступа)

$ git clone https://github.com/odoo/enterprise.git

Примечание

** Репозиторий Enterprise git не содержит полного исходного кода Odoo **. Это всего лишь набор дополнительных дополнений. Основной код сервера находится в версии сообщества. Запуск версии Enterprise фактически означает запуск сервера из версии Community с опцией addons-path, установленной в папку с версией Enterprise. Вам нужно клонировать репозиторий Community и Enterprise, чтобы иметь работающую установку Odoo Enterprise.

Подготовить

питон

Для запуска Odoo требуется Python 3.6 или новее. Используйте предпочитаемый менеджер пакетов (homebrew, macports), чтобы загрузить и установить Python 3 на свой компьютер, если это еще не сделано.

Примечание

Если Python 3 уже установлен, убедитесь, что он версии 3.6 или выше, так как предыдущие версии не совместимы с Odoo.

$ python3 --version

Убедитесь также, что pip установлен для этой версии.

$ pip3 --version
PostgreSQL

Odoo использует PostgreSQL в качестве системы управления базами данных. Используйте postgres.app для загрузки и установки последней версии PostgreSQL.

По умолчанию единственным пользователем является postgres, но Odoo запрещает подключение как` postgres`, поэтому вам нужно создать нового пользователя PostgreSQL:

$ sudo -u postgres createuser -s $USER
$ createdb $USER

Примечание

Поскольку ваш пользователь PostgreSQL имеет то же имя, что и логин Unix, вы сможете подключиться к базе данных без пароля.

зависимости

Зависимости Odoo перечислены в файле needs.txt, расположенном в корне каталога сообщества Odoo. Большинство из них можно установить с помощью ** pip **.

Совет

Может быть предпочтительнее не смешивать пакеты модулей Python между различными экземплярами Odoo или с вашей системой. Вы можете использовать virtualenv для создания изолированных сред Python.

Перейдите к пути установки вашего сообщества Odoo (YourOdooCommunityPath) и запустите ** pip ** в файле требований:

$ cd /YourOdooCommunityPath
/YourOdooCommunityPath$ pip3 install -r requirements.txt

Предупреждение

Не-Python-зависимости должны быть установлены с помощью менеджера пакетов:

  1. Загрузите и установите ** Инструменты командной строки **:

    $ xcode-select --install
  2. Загрузите и установите менеджер пакетов по вашему выбору (homebrew, macports).
  3. Установите не Python зависимости.
