Предупреждение

Этот урок требует хорошего знания Odoo. Пожалуйста, обратитесь к :ref:`basic tutorial <howto/base>`сначала, если нужно.

Начиная с версии 13.0, пользователь может войти в несколько компаний одновременно. Это позволяет пользователю получать доступ к информации от нескольких компаний, а также создавать / редактировать записи в среде нескольких компаний.

Если не обрабатывается правильно, это может быть источником многих противоречивых поведений нескольких компаний. Например, пользователь, вошедший в обе компании A и B, может создать заказ на продажу в компании A и добавить в него продукты, принадлежащие компании B. Только когда пользователь выйдет из компании B, ошибки доступа возникнут для заказа на продажу.

Для правильного управления поведением нескольких компаний ORM Odoo предоставляет несколько функций:

  • Company-dependent fields
  • Multi-company consistency
  • Default company
  • : Ссылка: `Просмотров <howto/company/views> `
  • Security rules

Зависимые от компании поля

Когда запись доступна из нескольких компаний, мы должны ожидать, что для данного поля будут назначены разные значения в зависимости от компании, из которой установлено значение.

Чтобы поле одной и той же записи поддерживало несколько значений, оно должно быть определено с атрибутом company_dependent, установленным в` True`.

from odoo import api, fields, models

class Record(models.Model):
    _name = 'record.public'

    info = fields.Text()
    company_info = fields.Text(company_dependent=True)
    display_info = fields.Text(string='Infos', compute='_compute_display_info')

    @api.depends_context('force_company')
    def _compute_display_info(self):
        for record in self:
            record.display_info = record.info + record.company_info

Примечание

Метод _compute_display_info декорирован с помощью метода 'depen_context (' force_company ')` (см .: attr: ~ odoo.api.depends_context), чтобы обеспечить пересчет вычисляемого поля в зависимости от принудительной / текущей компании (контекст` force_company`) ключ или self.env.company).

Когда поле, зависящее от компании, читается, текущая компания используется для получения его значения. Другими словами, если пользователь вошел в компании A и B с A в качестве основной компании и создал запись для компании B, значения зависимых от компании полей будут равны компании A.

Чтобы прочитать значения зависимых от компании полей, установленных из другой компании, отличной от текущей, в контекстном ключе force_company должен быть указан идентификатор нужной компании.

# Accessed as main company (self.env.company)
val = record.company_dependent_field

# Accessed as desired company (company_B)
val = record.with_context(force_company=company_B.id).company_dependent_field

Многопрофильность

Когда запись делится между несколькими компаниями посредством поля company_id, мы должны позаботиться о том, чтобы ее нельзя было связать с записью другой компании через поле реляционных данных. Например, мы не хотим, чтобы заказ на продажу и его счет принадлежали разным компаниям.

Чтобы обеспечить согласованность действий нескольких компаний, вы должны:

  • Установите атрибут класса _check_company_auto в` True`.
  • Определите реляционные поля с атрибутом check_company, установленным в` True`, если их модель имеет поле company_id.

Для каждого create() и create() будут выполняться автоматические проверки для обеспечения согласованности записи в нескольких компаниях.

from odoo import fields, models

class Record(models.Model):
    _name = 'record.shareable'
    _check_company_auto = True

    company_id = fields.Many2one('res.company')
    other_record_id = fields.Many2one('other.record', check_company=True)

Примечание

Поле company_id не должно быть определено с помощью` check_company = True`.

Model._check_company(fnames=None)[исходный код]

Проверьте компании значения указанных названий полей.

Параметры
fnames (list) – имена реляционных полей для проверки
Исключение
UserError – если значение company_id значения какого-либо поля отсутствует в` [False, self.company_id] (или self` if res_company).

Для res_users реляционных полей, проверяет, находится ли звукозаписывающая компания в полях` company_ids`.

Пользователь с основной компанией A, имеющий доступ к компаниям A и B, может быть назначен или связан с записями в компании B.

Предупреждение

Функция check_company выполняет строгую проверку! Это означает, что если запись не имеет company_id (то есть поле не является обязательным), она не может быть связана с записью, для которой установлен` company_id`.

Примечание

Если домен не определен в поле и для check_company установлено значение` True`, добавляется домен по умолчанию: `['|', '(' company_id ',' = ', False), (' company_id ',' = ', company_id)] `

Компания по умолчанию

Когда поле 'company_id` является обязательным для модели, рекомендуется установить компанию по умолчанию. Это облегчает процесс настройки для пользователя или даже гарантирует его действительность, когда компания скрыта от просмотра. Действительно, компания обычно скрыта, если у пользователя нет доступа к нескольким компаниям (т. Е. Когда у пользователя нет группы base.group_multi_company).

from odoo import api, fields, models

class Record(models.Model):
    _name = 'record.restricted'
    _check_company_auto = True

    company_id = fields.Many2one(
        'res.company', required=True, default=lambda self: self.env.company
    )
    other_record_id = fields.Many2one('other.record', check_company=True)

Взгляды

Как указано в above, компания обычно скрыта от просмотра, если пользователь не имеет доступа к нескольким компаниям. Это проверено с группой base.group_multi_company.

Правила безопасности

При работе с записями, которые распространяются между компаниями или ограничиваются одной компанией, мы должны следить за тем, чтобы у пользователя не было доступа к записям, принадлежащим другим компаниям.

Это достигается с помощью правил безопасности, основанных на company_ids, который содержит текущие компании пользователя (компании, которые пользователь проверил в виджете для нескольких компаний).

<!-- Shareable Records -->
<record model="ir.rule" id="record_shared_company_rule">
    <field name="name">Shared Record: multi-company</field>
    <field name="model_id" ref="model_record_shared"/>
    <field name="global" eval="True"/>
    <field name="domain_force">
        ['|', ('company_id', '=', False), ('company_id', 'in', company_ids)]
    </field>
</record>

<!-- Company-restricted Records -->
<record model="ir.rule" id="record_restricted_company_rule">
    <field name="name">Restricted Record: multi-company</field>
    <field name="model_id" ref="model_record_restricted"/>
    <field name="global" eval="True"/>
    <field name="domain_force">
        [('company_id', 'in', company_ids)]
    </field>
</record>
